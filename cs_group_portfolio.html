Context & Problem

Maritime surveillance is a critical national security challenge. Monitoring coastal zones, detecting irregular vessel movements, and transmitting structured intelligence to command centers requires automated, real-time solutions that work reliably in degraded conditions — night, fog, low contrast infrared imagery.

The mission was to design a complete video processing system capable of automatically detecting vessels in both infrared (IR) and visible-light video streams, classifying them by type, tracking each target over time, and exporting structured trajectory data for integration into operational command-and-control tools.

The project was treated as a production-grade R&D prototype — real annotated datasets, full IVVQ process, documented architecture, and milestone deliveries.

AI Processing Pipeline

The system processes raw video frames through a four-stage pipeline running in a multithreaded Python architecture, with GPU acceleration throughout.

Each stage runs in a dedicated thread. A "Last Frame Only" strategy prevents queue buildup, keeping end-to-end latency under 100 ms per frame on GPU. The full pipeline — read + YOLO + CNN + Deep SORT — sustains 30 fps without blocking the UI.

Three classification architectures were compared — MobileNetV3, ResNeXt50, and Vision Transformer (ViT). The ViT consistently outperformed CNNs, reaching ~97% accuracy on both IR and visible domains. Key Technical Achievements

🧵
Multithreaded real-time architecture

Designed a 4-thread pipeline (read / detect / classify / track) with shared memory queues. The "Last Frame Only" strategy eliminates accumulation lag, sustaining 30 fps output without blocking the user interface.

🤖
Custom IR model training pipeline

Built a full training pipeline for both IR and visible domains. Handled class imbalance with automatic weight computation. Fine-tuned YOLOv8 (500 epochs, 4536 IR images) and four classification architectures, reaching up to 97.18% accuracy.

📦
Modular IHM with dynamic model loading

The GUI supports runtime model swapping — any .pth + .py classifier file dropped in the classifiers/ folder is auto-detected. Detection threshold, camera parameters (azimuth, FOV, elevation), and tracker embedder are all configurable without restarting.

🗺️
Structured JSON export for C2 integration

Each tracked vessel is exported with ID, class, subclass, confidence, and full per-frame trajectory (x, y, direction angle). Format was designed for direct ingestion by an internal defense command-and-control platform.

Takeaways

This project was my first full end-to-end AI engineering mission in a production-constrained defense environment — from raw dataset to deployed GUI. The key lessons:

Architecture beats raw model performance. Choosing the right threading strategy and queue management had more impact on usability than squeezing an extra 0.5% accuracy from the classifier.

Dataset quality is everything in IR. Infrared imagery is noisy, low-contrast, and domain-specific — standard pretrained weights transfer poorly. Rigorous data collection, cleaning, and augmentation were what made the 97% ViT accuracy possible.

Modularity pays off immediately. Designing the GUI to load any classifier dynamically let me benchmark 4 architectures without touching core code — saving days of integration work.

Defense context adds real constraints. Working toward C2 integration meant the JSON schema, latency targets, and IVVQ documentation were non-negotiable requirements, not optional. That discipline shaped every technical decision.
